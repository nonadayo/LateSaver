ここではGithubを使った作業の流れを書いていくよ
全体の流れは以下のような感じ(※は読まなくても作業できる。少し詳しく知りたい人と作業がうまくいかない人向け)
------------------------------------------------------------------------------------------------------

1. 手元のファイル群を最新に更新
    git pull をVScodeのターミナルで実行
    ※githubが有効になってるディレクトリにいなければ実行しても意味ない
    ※githubが有効になっているのは .gitフォルダがある階層以下のディレクトリすべて
    ※.gitフォルダは git init もしくは git clone したディレクトリに自動作成されている


2. 各自ファイルを編集・保存する
    ※"./manuals/お約束.txt" を読んでくれるとチーム制作がスムーズかも


3. 編集内容を保存する
    VScodeのGithub拡張機能を使ってコミットする。
        コミットしたいものをステージする (Changesの欄に編集内容があるので、カーソルを合わせて＋ボタンを押す)
        ステージに上げたものを、変更内容を軽くメッセージに残したうえでコミットボタンを押す
    ※ commitしたときに行われるのはローカルへの保存である。
    ※ ただローカルに変更したファイルとかを全部保存してたら容量がえぐいことになる
    ※ gitでは差分だけを保存することで容量を削減している


4. もう一度手元のファイルを最新にする
    git pull をVScodeのターミナルで実行
    ※2の作業をしていると、その間に誰かが新しく内容を変えているかもしれない。
    ※その人の変更を古いデータで上書きしないようにアップロードする前にもう一度更新する
    ※Conflict といって更新した結果自分の編集内容と競合がおきてエラーが起きる可能性がある。
    ※この場合は手動で解決するが、分からなければ板倉に連絡ください


5. 編集内容をリモートのGithubプロジェクトにアップする
    VScodeのGithub拡張機能を使ってpushする。
        すべての内容をコミットし終わると、コミットボタンだったところがasync changesに変わるのでそれを押す


6. 作業終了
